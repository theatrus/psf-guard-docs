Getting Started

If N.I.N.A. already tracks your images, the quickest setup is the desktop app: add the Target Scheduler database and its image folders in Settings, then open a project. You do not need the CLI, an import, or a second database for this path.

Back up your Target Scheduler database before first use. Grading writes to it. Import and sync make wider changes, but both provide a preview before they write.

The simple path: open one database

  1. Install the desktop app for your platform.
  2. Open Settings → Add Database.
  3. Select the Target Scheduler SQLite file and add the folders that hold its FITS images. On Windows, the usual database path is %LOCALAPPDATA%\NINA\SchedulerPlugin\schedulerdb.sqlite.
  4. Choose a project on Overview, then open Images or Sequence.
PSF Guard does not copy your FITS files. The image folders can stay read-only, but the scheduler database must be writable if you want to save grades or planning changes.

Install the desktop app

Grab the installer for your platform from the latest release:

PlatformAsset
Windows x64PSF.Guard_<version>_x64_en-US.msi or the NSIS -setup.exe
macOS (Apple Silicon)PSF.Guard_<version>_aarch64.dmg
Linux x64PSF.Guard_<version>_amd64.deb or .AppImage

Install, launch, and point the settings panel at your scheduler database and image directories. The Windows NSIS installer also installs a console psf-guard-cli.exe and adds it to your user PATH, so the full CLI works from any terminal.

Review the first project

  1. Use Overview to check projects, targets, completion, and file status.
  2. Open Images. Filter by status or channel, move with the arrow keys, and press Space to build a selection.
  3. Accept or reject frames, or open Sequence to review a whole session.
  4. Run Scan Quality only when you want fresh pixel-based cloud, obstruction, and pointing evidence.

Continue with The Grader UI. Plate solving, satellite tracks, stack previews, import, and sync are optional guides in the sidebar; they are not setup requirements.

Server, NAS, and CLI setups

CLI Use the command line only when the setup needs itThe desktop app covers the common one-database review flow. The CLI adds server hosting, batch screening, import, export, and automation. Full reference →

Docker on a Linux server or NAS

docker run -d -p 3000:3000 \
  -v /path/to/schedulerdb.sqlite:/data/database.sqlite \
  -v /path/to/images:/images:ro \
  ghcr.io/theatrus/psf-guard:latest

Then open http://localhost:3000/. The database mount must be writable — grading writes back to it. To tune the port, cache, or preview pre-generation, mount a TOML config at /data/config.toml and append server --config /data/config.toml /data/database.sqlite /images (see Configuration).

Standalone CLI binaries

These version-independent links point at the latest release:

PlatformDownload
Linux x64psf-guard-linux-x64
macOSpsf-guard-macos-x64
Windows x64psf-guard-windows-x64.exe
# Linux / macOS
chmod +x psf-guard-*
./psf-guard-linux-x64 server schedulerdb.sqlite /path/to/images/
:: Windows — back up the DB first, then serve it
copy "%LOCALAPPDATA%\NINA\SchedulerPlugin\schedulerdb.sqlite" schedulerdb-backup.sqlite
psf-guard-windows-x64.exe server "%LOCALAPPDATA%\NINA\SchedulerPlugin\schedulerdb.sqlite" C:\path\to\images

Then open http://localhost:3000/.

Fedora RPM

Releases attach prebuilt RPMs for Fedora 43 and 44:

sudo dnf install ./psf-guard-*.fc44.x86_64.rpm

The package installs the CLI/server and a psf-guard.service systemd unit. To build your own, see packaging/rpm/README.md.

Build from source 
git clone https://github.com/theatrus/psf-guard.git
cd psf-guard
cargo build --release
./target/release/psf-guard server schedulerdb.sqlite /path/to/images/

You need Rust and Node.js/npm; the React frontend is embedded at build time. Image processing uses the published pure-Rust Seiza crates.

Two-machine setups

Sync Keep the telescope database and a grading copyPull full projects and captures to the grading machine. Push only edited plans and grades back to the telescope. Sync workflow →

Do not set up sync when the desktop app can open the telescope database directly. Sync is for two separate database files that must exchange state.

Current image limits

PSF Guard reads monochrome FITS files. It does not yet debayer color or OSC frames.
