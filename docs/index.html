Getting Started

Install PSF Guard, then build an image catalog from FITS folders or open an existing N.I.N.A. Target Scheduler database. Both paths keep the image files where they are and need no command line.

1. Install PSF Guard

The desktop app is the shortest path on Windows, macOS, and Linux. The installation guide also covers Docker, standalone CLI binaries, Fedora RPMs, and source builds.

Like Lightroom Classic, but for astrophotography data: PSF Guard catalogs files where they already live. It uses the Target Scheduler database structure for its image map, plans, and grades, but does not require Target Scheduler.

3. Start a catalog

Choose Settings → New Database from Images, select the FITS folders, and review the header-only import before creating the catalog. PSF Guard groups the images and derives plans without waiting for pixel analysis. See Add Images & Plan for the full flow.

Or open a Target Scheduler database

Back up the Target Scheduler database before the first write. Grading updates it. Import and sync preview wider changes before applying them.
  1. Open Settings → Add Database.
  2. Select the Target Scheduler SQLite file and add the folders that hold its FITS images. On Windows, the usual database path is %LOCALAPPDATA%\NINA\SchedulerPlugin\schedulerdb.sqlite.
  3. Choose a project on Overview, then open Images or Sequence.
PSF Guard does not copy your FITS files. The image folders can stay read-only, but the scheduler database must be writable if you want to save grades or planning changes.

4. Review the first project

  1. Use Overview to check projects, targets, completion, and file status.
  2. Open Images. Filter by status or channel, move with the arrow keys, and press Space to build a selection.
  3. Accept or reject frames, or open Sequence to review a whole session.
  4. Run Scan Quality only when you want fresh pixel-based cloud, obstruction, and pointing evidence.
  5. When grading is done, export the Accepted lights from Overview into a clean stacking tree or ZIP.

Continue with The Grader UI. Plate solving, satellite tracks, stack previews, export, import, and sync are optional guides in the sidebar; they are not setup requirements.

Do not set up sync when the desktop app can open the telescope database directly. Sync is for two separate database files that must exchange state.

Current image handling

PSF Guard reads monochrome FITS directly. It also debayers raw one-shot-color FITS files with a recognized BAYERPAT header, then uses luminance for grading and quality measurements. The single-frame grader does not yet render an OSC exposure in full color.
